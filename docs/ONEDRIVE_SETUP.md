# Save this project to OneDrive

Use these steps so the project lives in OneDrive and syncs to your desktop and other devices.

**For OneDrive + Git together (work on desktop and use version control):** see **[ONEDRIVE_AND_GIT_SETUP.md](./ONEDRIVE_AND_GIT_SETUP.md)**.

---

## 1. Find your project and OneDrive folder

- **Current project folder:** The folder that contains this file (e.g. `OutofRoutebuddy` or `OutOfRoutebuddy`). On your machine it’s likely something like:
  - `C:\Users\brand\OutofRoutebuddy`
  - or `C:\Users\brand\Documents\OutofRoutebuddy`
- **OneDrive folder:** Usually one of:
  - `C:\Users\brand\OneDrive`
  - `C:\Users\brand\OneDrive - Personal`
  - `C:\Users\brand\OneDrive - [Company name]`

---

## 2. Move or copy the project into OneDrive

**Option A – Move (recommended if this is the only copy)**  
1. Close Cursor and any other apps using the project (e.g. Android Studio).  
2. In File Explorer, cut the whole project folder (Ctrl+X).  
3. Open your OneDrive folder.  
4. Paste (Ctrl+V).  
5. Reopen the project in Cursor: **File → Open Folder** and choose the project inside OneDrive.

**Option B – Copy (keep a second copy somewhere else)**  
1. Copy the whole project folder (Ctrl+C).  
2. Open your OneDrive folder and paste (Ctrl+V).  
3. Open the **OneDrive copy** in Cursor and use that from now on so it stays in sync.

---

## 3. Optional: Exclude build folders from OneDrive (faster sync, fewer locks)

OneDrive will sync everything by default. The `build/` and `.gradle/` folders are large and regenerated by Gradle. You can exclude them so OneDrive doesn’t sync them:

- **OneDrive:** Right‑click the OneDrive icon in the taskbar → **Settings** → **Sync and backup** → **Manage backup** (or **Choose folders**), or use **Settings → Sync** to exclude specific folders.  
- Exclude the **`build`** and **`.gradle`** folders inside your project if your OneDrive app supports folder exclusions.  
- If it doesn’t, you can leave them; sync will just be heavier and you may see “locked” messages if Android Studio is building while OneDrive is syncing.

---

## 4. Open the project on your desktop

- After the project is in OneDrive, wait for sync to finish (check the OneDrive icon).  
- On your desktop PC: open **File Explorer → OneDrive** and find the project folder.  
- Install **Cursor** (or Android Studio) on the desktop, then **File → Open Folder** and select that project folder inside OneDrive.

---

## 5. Checklist

- [ ] Project folder is inside your OneDrive folder (not just a shortcut).  
- [ ] OneDrive shows the folder as synced (green checkmark or “Synced”).  
- [ ] You opened the project in Cursor from the OneDrive path (so edits save there).  
- [ ] On desktop you can open the same folder from OneDrive in Cursor.

Once that’s done, the project is saved to OneDrive and will sync to your desktop and other devices.
