# Improvement Loop — Variants

**Purpose:** Define loop variants so user can choose depth. Research: 30–90 min typical; 2-hour valid for comprehensive.

**References:** [IMPROVEMENT_LOOP_RESEARCH_2025-03.md](./IMPROVEMENT_LOOP_RESEARCH_2025-03.md), [IMPROVEMENT_LOOP_ROUTINE.md](./IMPROVEMENT_LOOP_ROUTINE.md)

---

## Variants

| Variant | Duration | Trigger | Phases |
|---------|----------|---------|--------|
| **Quick** | 30 min | **GO quick** | 0.0 checkpoint, 0.1 research (5 min), 1.1 one quick win (10 min), pulse (5 min), 4.3 summary (5 min) |
| **Standard** | 90 min | **GO standard** | 0, 1, 2, 4 (skip or shorten Phase 3) |
| **Full** | 2 hours | **GO** | 0, 1, 2, 3, 4 (complete routine) |

---

## Quick (30 min) — Phase Subset

| Step | Time | Action |
|------|------|--------|
| 0.0a | 1 min | Read USER_PREFERENCES_AND_DESIGN_INTENT (get personal with the code) |
| 0.0 | 2 min | Pre-loop checkpoint |
| 0.1 | 5 min | Research (abbreviated: CRUCIAL, last summary) |
| 1.1 | 10 min | One quick win (dead code or doc link) |
| 1.4 | 5 min | Pulse check |
| 4.3 | 5 min | Summary (abbreviated) |

---

## Standard (90 min) — Phase Subset

| Phase | Time | Action |
|-------|------|--------|
| 0 | 15 min | Checkpoint, research, classify |
| 1 | 25 min | Quick wins, security, smoothness |
| 2 | 35 min | Test health, docs, sandboxing |
| 4 | 15 min | Lint, pulse, summary |

**Skip:** Phase 3 (UI polish) or shorten to 5 min (one string fix only).

---

## Full (2 hours) — Complete Routine

All phases 0–4 as defined in [IMPROVEMENT_LOOP_ROUTINE.md](./IMPROVEMENT_LOOP_ROUTINE.md).
