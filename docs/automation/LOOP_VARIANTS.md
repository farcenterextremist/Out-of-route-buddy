# Improvement Loop — Variants

**Purpose:** Define loop variants by scope. Complete tasks on todo lists. No fixed duration.

**References:** [IMPROVEMENT_LOOP_ROUTINE.md](./IMPROVEMENT_LOOP_ROUTINE.md), [LOOP_TIERING.md](./LOOP_TIERING.md)

---

## Variants

| Variant | Trigger | Scope |
|---------|---------|-------|
| **Light only** | **GO light** | Light tier only: verification, docs, brainstorm, debugging, populate tasks |
| **Light + Medium** | **GO** or **GO standard** | Light and Medium: quick wins, security, sandbox, test fix, dead code (1–2 items) |
| **Full** | **GO full** | All phases 0–4: research, quick wins, security, tests, sandbox, UI polish, summary |

---

## Light only

- Checkpoint, research, classify
- Execute Light tasks only (additive: docs, verification, brainstorm, debugging)
- Pulse, summary

---

## Light + Medium

- Checkpoint, research, classify
- Execute Light and Medium tasks
- One quick win, one security item, one sandbox improvement
- Pulse, summary

---

## Full

All phases 0–4 as defined in [IMPROVEMENT_LOOP_ROUTINE.md](./IMPROVEMENT_LOOP_ROUTINE.md).
