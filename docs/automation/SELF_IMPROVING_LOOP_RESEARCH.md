## Principles

| Principle | Application |
|-----------|-------------|
| **Single highest-impact change** | Per run, focus on one or two highest-impact items; avoid scattering. |
| **Light & Medium auto-implement** | Tier per LOOP_TIERING; implement without stopping. |
| **Heavy = document only** | Drastic loop improvements (routine changes, new phases, new loops) require human approval. |
| **Hub as single source** | Polished outputs live in hub/; next run reads them. |
| **Critique before deposit** | Ask: actionable? real artifacts? reusable? No padding. |
