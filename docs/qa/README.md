# QA

This folder holds **test strategy and test plans** owned by the QA Engineer.

- **TEST_STRATEGY.md** — What we test (unit vs integration vs UI), when, quality gates (when created).
- **TEST_PLAN_<feature>.md** — Per-feature test scenarios and how to run (e.g. Auto drive).
- **FAILING_OR_IGNORED_TESTS.md** — Tracking for failing/ignored tests (TripInputViewModelIntegrationTest, TripHistoryByDateViewModelTest, etc.); fix or document.
- **COORDINATOR_EMAIL_SMOKE.md** — Smoke test for send/read email (open line).
- **REGESSION_CHECKLIST.md** — Manual (or automated) checklist: Trip start/end, History, Statistics.

See `docs/agents/data-sets/qa.md` for the full data set.
